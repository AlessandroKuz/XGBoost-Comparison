XGBoost Overview

XGBoost is a state-of-the-art Regression and Classification Algorithm.

  • It is accurate, fast and easy to use.
    • Accuracy: Built in regularizations prevent the model from overfitting.
    • Speed: Thanks to parallelization and other optimizations it is able to handle massive datasets efficiently.
    • Ease-of-use: Implements both sklearn and native distributions.
  • Extra Features:
    • Easily allows missing and categorical data.
    • Has some characteristics similar to DL (Early stopping, Incremental training).

it builds on top of the idea of Ensemble methods.

It uses Boosting instead of Bagging, specifically it uses Gradient Boosting.

Ensemble Learning Methods

Ensemble Learning is a technique that bundles different models to achieve a greater outcome, making it more robust through “the power of big numbers”.

There are different types of techniques:

  • Simple Ensemble Techniques:
    • Max Voting (Hard or Soft)
    • Averaging
    • (Weighted Average)
  • Advanced Ensemble techniques
    • Bagging
    • Boosting

Bagging vs Boosting

Bagging (Bootstrap Aggregating):

  • Bagging creates multiple independent models, each trained on a random subset with replacement of the training data (bootstrap samples).
  • The final prediction is typically made by aggregating (averaging or voting) the predictions of all base models.
  • Models in bagging are trained independently of each other on sligthly different data and features.
  • Random Forest is a popular example.

Boosting:

  • Boosting focuses on sequentially training weak learners to adjust for the errors made by the ensemble so far.
  • The final prediction is a weighted sum of the individual predictions.
  • Boosting is adaptive; it adjusts its focus during training based on the difficulty of the examples, putting more emphasis on challenging instances.
  • AdaBoost and Gradient Boosting Machines (GBM) are popular examples.

A quick look into two examples:

Random Forest:

  • Built by a predifined number of independent Decision Trees who try to optimize Gini or Entropy indeces.
  • Trees are built in parallel and have the same weight/amount of say on the final prediction.
  • Each estimator is a full tree but it could (pre-)pruned to improve generalization.
  • Part of the data could be kept out as OOB data.

AdaBoost:

  • Built by sequential Weak Learners combined into a strong learner by Weighted Average/Voting.
  • Each stump has a different associated weigth thus a different amount of say, based on previous predictions.
    • Formula for say: 12log(1−TotalErrorTotalError)\frac{1}{2}log(\frac{1-Total Error}{Total Error})21​log(TotalError1−TotalError​)﻿.
    • The weights are updated via Weigth∗e±Say{Weigth}*e^{±{Say}}Weigth∗e±Say﻿ or by Resampling.
  • The algorithm keeps building estimators until it reaches either a predetermined accuracy or n estimators.

Instead of tweaking weights at every iteration we can use Gradient boosting.

Gradient Boosting and XGBoost

Gradient Boosting:

  • Gradient Boosting is a specific type of boosting algorithm that focuses on optimizing a loss function using gradient descent.
  • It optimizes a differentiable loss function by fitting each weak learner to the negative gradient of the loss function with respect to the model's predictions.
  • Weak learners are added sequentially, and each one is trained to predict the residuals of the ensemble.
  • Each weak learner contributes with a weight, usually determined by a learning rate.
  • Similar to boosting, Gradient Boosting is adaptive. It adjusts its focus during training based on the errors made by the ensemble so far.
  • Gradient Boosting Machines (GBM), including popular implementations like XGBoost, LightGBM and CatBoost.

XGBoost:

  • XGBoost is a specific implementation of the gradient boosting algorithm designed for speed and performance.
  • XGBoost incorporates additional features such as regularization terms in the objective function, making it more robust against overfitting.
  • It includes a "pruning" that removes splits that contribute little to the overall improvement and it usually limits the leaves of its weak learners to a range of 8 to 32.
  • Handling Missing and Categorical Data: XGBoost excels in handling missing and categorical data seamlessly, providing robust solutions for diverse datasets.
  • It incorporates characteristics similar to Deep Learning, such as early stopping and incremental training, enhancing its flexibility and efficiency.
  • XGBoost incorporates regularization terms, such as L1 (Lasso) or L2 (Ridge), to control the complexity of the individual trees and prevent overfitting. Additionally, it leverages parallelization for efficient training.
  • An essential feature of XGBoost is its ability to handle missing values during both training and prediction.
  • XGBoost provides a built-in feature importance score, aiding in feature selection.
  • XGBoost includes built-in cross-validation during training.
  • In regression tasks we usually start by taking the mean of the label, in classification we use the log(odds) the converted to probability.

XGB Optimizations
  • Regularization and preventing overfitting:
    • Pruning: Post-Pruning (Gamma) or Pre-Pruning (max_depth, …)
    • Integrated Cross Validation
    • Allows early stopping and incremental training
    • DART: Similar to Dropout in DL models
  • Parallelization and speed improvements:
    • CPU Cache
    • Uses Quantiles instead of average values
  • Extra features:
    • Shap when native implementation
    • Automatically handles missing data
    • Handles categorical data specifying enable_categorical
  • Hyperameters:
    • booster: gbtree, gblinear or dart
    • objective: reg, binary, multi
    • colsubsample_by*: tree, level, node
    • subsample
    • When booster=gbtree:
      • eta/learning_rate
      • gamma/min_split_loss
      • max_depth
      • min_child_weight
      • max_delta_step
    • When gblinear:
      • alpha/reg_alpha: L2/Squared/Ridge
      • lambda/reg_lambda: L1/Absolute/Lasso

XGBoost:

For regression, it begins with an initial prediction and calculates a quality score for residuals, using a threshold to split residuals into two groups. The gain from splitting is evaluated, and the process repeats to build a tree with a default depth of 6. Pruning is based on the gain values, and the output value is determined using a combination of initial prediction, learning rate, and residuals.

In classification, XGBoost starts with a 0.5 prediction, and the process involves calculating similarity scores for residuals, choosing thresholds, and evaluating gains. Pruning, similar to regression, is based on a threshold for the minimum number of residuals in each leaf (cover). The output value for each leaf is determined using a specific formula, and predictions are iteratively calculated until reaching the specified number of trees or minimal residuals.

GAMMA: minimum treshold (of gain) to reach in order to keep the node - if smaller than treshold the the node gets pruned; it starts at the lowest branch, if it gets pruned it goes up until there's no more pruning to be done.

Uses the greedy algorithm to build trees (because the split is dependent on gain and it does not worry about how the leaves will be split later). It makes a decision without looking ahead to see if it is the absolute best choice in the long term. It builds the tree relatively quickly

Use quantiles as candidate thresholds to split the observations

Increasing the number of quantiles will provide more accurate predictions

To increase speed quantiles are tested in xgboost. The greedy algorithm uses about 33 quantiles

Sketch algorithms are used to find the approximate answers – Weighted quantile sketch. The sum of the weights in each quantile will be the same. The instances will have different weights according to the cover metric

The advantage of using the weighted quantile sketch is that we get smaller quantiles when we need them

It uses parallel learning to split up the dataset so that multiple computers can work on it at the same time

Sparsity-aware split finding tells us how to build trees with missing data

Cache-aware access – XGBOOST puts gradients and hessians in the cache so that it can rapidly calculate similarity scores and output values

Blocks for out of core computation – The data stored in the hard drive is compressed. xgboost will spend a little bit of CPU time uncompressing the data. This way it avoids spending a lot of time accessing the hard drive

If more than one hard drive is available, it uses sharding for faster access
